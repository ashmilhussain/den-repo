---
name: Documentation Style Guide
description: Voice, tone, and formatting standards for all documentation
---

Voice and tone:
- Active voice: "Click Save" not "Save should be clicked"
- Second person: "You can" not "The user can"
- Present tense: "The system returns" not "The system will return"
- Friendly but not casual — professional without jargon

Formatting:
- Headers: sentence case (not Title Case) except for proper nouns
- Code: always in `code blocks`, never in italics
- UI elements: **bold** ("Click **Save**")
- Warnings: use admonition blocks (> Warning: ...)

Sentence starters to avoid:
- "Simply..." (nothing is simple to someone learning)
- "Just..." (same problem)
- "Obviously..." (if it were obvious, they wouldn't need docs)
- "As you know..." (condescending)
