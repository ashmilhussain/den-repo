Agile/Scrum practices for team coordination:

## Sprint structure
- Sprint length: 1-2 weeks (consistent)
- Sprint planning: beginning of sprint — pick items from backlog, estimate, commit
- Daily standup: what I did, what I'm doing, any blockers (keep under 15 minutes)
- Sprint review: end of sprint — demo what was built to stakeholders
- Sprint retrospective: end of sprint — what went well, what didn't, action items

## Story format
```
As a [user type], I want to [action] so that [outcome].

Acceptance criteria:
- [ ] Criterion 1
- [ ] Criterion 2
- [ ] Criterion 3

Story points: [estimate]
```

## Estimation
- Use relative story points (1, 2, 3, 5, 8, 13)
- 1 point = trivial change, well-understood
- 8+ points = should be broken into smaller stories
- 13 points = too large, must be decomposed before sprint planning

## Backlog management
- Product Owner prioritises the backlog
- Top of backlog should be refined and ready (acceptance criteria, estimates)
- Bottom of backlog is rough and unestimated — that's fine
- Remove items that have been in the backlog for 3+ months without movement

## Definition of Done
- Code complete and passing tests
- Code reviewed and approved
- Deployed to staging
- Acceptance criteria verified
- Documentation updated (if applicable)
